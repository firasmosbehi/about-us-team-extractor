# Contributing

Thanks for taking the time to contribute.

## Quick Start
- Create an issue describing the change (bug/feature) before starting work.
- Fork the repo and create a feature branch.
- Keep PRs small and focused.

## Development
- Node.js: `>= 20`
- Install: `npm install`
- Run locally: `npm start` (expects Apify-style input when running via `apify run`)
- Tests: `npm test`

## Code Style
- Formatting: `npm run format`
- Lint: `npm run lint`

## Pull Requests
- Describe what changed and why.
- Include tests for bug fixes and non-trivial logic.

## Reporting Bugs
Open an issue with:
- Steps to reproduce
- Expected vs actual behavior
- Logs and relevant input (redact secrets)
